# Face Muse

Face Muse 是面向美妆新手的移动端 AI 妆容体验原型。项目将实时妆容指导、眉眼校准、收藏妆上脸预演、公开妆容激励和化妆师服务匹配整合进同一个 feed 浏览场景，让用户在刷到内容时直接获得可执行建议并完成下一步操作。

当前版本为高保真静态前端实现，无后端请求、无构建链、无真实模型调用。AI 能力通过页面内的状态数据、交互反馈和视觉提示进行模拟，适合用于产品演示、评审走查和交互验证。

## 项目亮点

- 刷到即用：将 AI 建议嵌入 feed 卡片，减少从内容到工具的跳转成本。
- 实时指导：围绕光线、步骤、眉峰、眉尾、睫毛角度等状态给出即时提示。
- 上脸预演：把收藏妆容拆成底妆、眼影、腮红、唇线等练习步骤。
- 内容激励：通过公开妆容、一键使用和美力值榜单模拟内容供给闭环。
- 服务匹配：结合历史妆造档案、距离、评分和风格偏好推荐化妆师或工作室。
- 轻量实现：使用 HTML、CSS、原生 JavaScript 和本地 GSAP 资源，直接打开即可体验。

## 页面结构

| 文件 | 页面 | 说明 |
| --- | --- | --- |
| `index.html` | 拼接展示入口 | 使用 iframe 聚合五个移动端页面，支持按钮、滚轮、键盘和拖拽翻页。 |
| `1.html` | 实时妆容指导 | 展示当前步骤、光线校准、定位目标和语音提示状态。 |
| `2.html` | 眉眼校准 | 提供定位、补空、复查三种状态下的即时指导。 |
| `3.html` | 收藏妆上脸预演 | 在参考图和上脸预演之间切换，并生成四步练习。 |
| `4.html` | 公开妆容与美力榜 | 展示连续登录、累计登录、公开妆容使用和拉新奖励。 |
| `5.html` | 化妆师与妆造档案 | 根据偏好、距离和评分模拟工作室推荐。 |

## 本地预览

最简单的方式是直接打开根目录的 `index.html`。

如果希望用本地静态服务预览，可以在项目根目录执行：

```bash
python -m http.server 4175
```

然后访问：

```text
http://127.0.0.1:4175/index.html
```

五个单页也可以分别访问：

```text
http://127.0.0.1:4175/1.html
http://127.0.0.1:4175/2.html
http://127.0.0.1:4175/3.html
http://127.0.0.1:4175/4.html
http://127.0.0.1:4175/5.html
```

## 技术实现

- 前端基础：HTML、CSS、原生 JavaScript。
- 动效能力：本地 `gsap.min.js` 和页面内轻量动效逻辑。
- 资源策略：使用本地 WebP、PNG、JPG 视觉资产，关键图片设置明确尺寸和加载策略。
- 交互方式：按钮点击、键盘方向键、鼠标滚轮、pointer 拖拽、状态按钮切换。
- 无障碍处理：页面使用 `aria-label`、`aria-pressed`、`aria-live` 等属性同步交互状态。
- 性能记录：仓库保留了既有 Lighthouse 报告文件，用于追踪页面性能与可访问性检查结果。

## 核心交互

1. 在 `index.html` 中横向切换五个 feed 页面。
2. 在实时指导页点击“重新校准”或“开始下一步”，查看步骤和提示文案变化。
3. 在眉眼校准页切换“定位 / 补空 / 复查”，查看扫描标签和指导文案联动。
4. 在试妆预演页切换参考图、自拍视频和练习步骤，查看适配度与流程状态。
5. 在美力榜页切换好友榜、总榜、连续登录和累计登录维度。
6. 在化妆师页切换匹配、工作室和档案状态，查看推荐说明变化。

## 已知边界

- 当前为静态原型，尚未接入真实摄像头、真实人脸关键点检测、试妆生成模型或后端推荐服务。
- 页面代码以比赛/演示交付为主，多个 HTML 文件内存在相似的 feed 布局和状态逻辑。
- 后续进入生产化阶段时，可优先抽取共享 feed 组件、统一状态模型，并将 AI 或推荐服务输出收敛为稳定的数据结构。
